  • Iot Inspector

    Languages: Python

    Skills/Frameworks/Libraries: ZeroConf, Network Device Detection, MDNS & UPNP Data Collection

    In my research, I aimed to address a significant flaw in the IoT Inspector, an open-source application designed to identify Internet of Things (IoT) devices on local networks. This application, which is vital for cybersecurity, was failing to properly recognize MDNS, SSDP, and UPNP devices, which constitute a major portion of IoT devices. On analyzing the source code, I identified that the root of the issue lay in the ‘netdisco’ library, which was deprecated. My approach to solving this problem combined in-depth software analysis with practical experimentation. Using a Raspberry Pi and a local router, I thoroughly tested the current identification processes and realized that the library was overly complicated and limited in its device detection, focusing only on specific IoT devices for which it had templates.

    To address these limitations, I devised two distinct solutions for SSDP/UPNP and MDNS devices. For SSDP/UPNP, I leveraged patterns within the information strings produced by the ssdp.scan() function, identifying a common substring that appeared in all UPNP devices. By creating a child class that inherited functionality from SSDPDiscoverable, I was able to redefine the device identification process and improve detection. For MDNS, I circumvented the complexities of the 'netdisco' library by integrating the ZeroConf library directly into the IoT Inspector application, which enabled efficient identification of MDNS devices. This hands-on approach not only utilized my software development and debugging skills but also underscored the importance of adaptability and innovative thinking in resolving complex technical challenges.
